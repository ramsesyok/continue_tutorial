    第1章 はじめに

    この章で学ぶこと¶

    • Continue とは何か、どのような機能を持つ AI コーディングアシスタントかを理解する
    • なぜオンプレミス・エアギャップ環境で Continue を使うことが有効なのかを理解する
    • このチュートリアルの対象読者と、始める前に必要な前提条件を把握する
    • 読み終えたあとに何ができるようになるかを確認する
    • チュートリアル全体の章構成を把握し、自分の目的に応じた読み方を選ぶ

    Continue とは何か¶

    AI コーディングアシスタントとしての位置付け¶

    Continue（コンティニュー）は、VS Code に追加できる AI コーディングアシスタントの拡張機能です。コードの質問に答えたり、補完候補を提示したり、既存のコードを書き換えたりと、開発作業のさまざまな場面で AI を活用できます。

    Continue の大きな特徴は、接続する LLM（大規模言語モデル）を自由に差し替えられる点です。クラウド上の外部サービスだけでなく、社内ネットワーク内で動かすローカル LLM にも対応しています。本チュートリアルでは、この「ローカル LLM に接続する」構成を前提として説明します。

    Continue が提供する主な機能は次の 4 つです。

    • Chat（チャット）: AI に質問したり、コードの説明を求めたりする対話機能
    • Autocomplete（オートコンプリート）: 入力中のコードに対して続きを自動補完する機能
    • Edit（エディット）: 選択したコードを指示に従って書き換える機能
    • Agent（エージェント）: ファイルの作成・編集・コマンド実行などを AI が自律的に行う機能

    各機能の詳しい使い方は、第 6 章以降でそれぞれ丁寧に解説します。

    他のツールとの違い¶

    GitHub Copilot や Amazon CodeWhisperer などのクラウド型 AI コーディングツールは、入力したコードをクラウド上のサーバへ送信して補完候補を生成します。便利な反面、コードが社外のサーバへ送出されるという点で、機密性の高い開発環境では利用を制限されることがあります。

    Continue はこの問題を解消できます。接続先を社内のローカル LLM に設定すれば、コードはネットワークの外へ一切出ません。また、LLM の種類・バージョン・設定をチームで一元管理できるため、組織として AI 利用の品質と安全性を統制しやすくなります。

    ハルシネーションについて

    AI コーディングアシスタントは非常に便利な道具ですが、もっともらしく見えても誤った情報やコードを出力することがあります（これを「ハルシネーション」と呼びます）。AI の回答は必ず自分の目で確認し、鵜呑みにしないことが重要です。この点はチュートリアル全体を通して繰り返し強調します。

    なぜオンプレミス・エアギャップ環境で使うのか¶

    セキュリティ要件とコンプライアンス¶

    金融・医療・防衛・製造など、多くの業界では情報セキュリティ規程や法令により、ソースコードや業務データをインターネット経由で外部に送信することが禁止または制限されています。こうした環境では、クラウド型のコーディングアシスタントをそのまま導入することが難しい場合があります。

    オンプレミス・エアギャップ環境（外部ネットワークから物理的・論理的に遮断された環境）で Continue を動かすと、コードや質問の内容が社外へ流出するリスクをゼロに近づけられます。監査やセキュリティレビューの際にも、「AI ツールを使っているが、データは社内に閉じている」と説明できる根拠を作りやすくなります。

    外部通信の遮断確認は必須です

    ソフトウェアの設定だけでは外部通信を完全に保証できない場合があります。第 4 章では、テレメトリ（使用状況の送信）を含む外部通信の遮断設定と、実際に通信が発生していないことを確認する手順を詳しく説明します。セキュリティ要件の厳しい環境では、必ず第 4 章の手順を実施してください。

    ローカル LLM との組み合わせがもたらすメリット¶

    本チュートリアルでは、Continue を次の構成で利用します。

    • VS Code: 開発者が日常的に使う統合開発環境
    • Continue 拡張: VS Code に追加する AI コーディングアシスタント
    • ローカル LLM: 社内ネットワーク内で稼働する言語モデル（Ollama・vLLM・LM Studio など）

    この構成の主なメリットは次の通りです。

    • コードや質問内容が社外に出ない
    • インターネット接続が不要なため、完全エアギャップ環境でも利用できる
    • 使用する LLM の種類・バージョンをチームで固定・管理できる
    • クラウドサービスの障害や仕様変更に左右されない

    本チュートリアルの前提

    ローカル LLM のインストールおよび Continue からの接続設定（config.yaml の記述）は、すでに完了している前提で説明を進めます。まだ設定が済んでいない場合は、担当のシステム管理者または IT 部門にご確認ください。

    対象読者と前提知識¶

    このチュートリアルを想定している読者¶

    本チュートリアルは、AI コーディングアシスタントを初めて使うソフトウェア開発者を主な読者として想定しています。具体的には、次のような方を念頭に置いています。

    • VS Code を使ってコードを書いている
    • GitHub Copilot や ChatGPT などの AI ツールをコーディングに使ったことがない、または使い始めたばかり
    • 社内のオンプレミス・エアギャップ環境で開発している
    • AI ツールの安全な使い方に関心がある

    プログラミング言語の種類は問いません。Python・Java・TypeScript・Go など、どの言語を使っていても活用できます。

    始める前に用意しておくもの¶

    本チュートリアルを進めるにあたり、次の条件が満たされていることを確認してください。

    • VS Code がインストールされている（バージョン 1.85 以上。詳細は 第 2 章 で確認します）
    • Continue 拡張がインストールされている（オフラインでの導入手順は 第 3 章 で説明します）
    • ローカル LLM が社内ネットワーク内で稼働しており、Continue からの接続設定が完了している

    上記の条件がまだ整っていない場合は、第 2 章 から順番に読み進めることをお勧めします。

    このチュートリアルを読み終えると¶

    本チュートリアルを読み終えると、次のことができるようになります。

    • Chat を使って、コードの書き方や設計の考え方を AI に質問できる
    • Autocomplete を活用して、タイプ量を減らしながらコーディングできる
    • Edit を使って、既存コードのリファクタリングやバグ修正を AI と協力して行える
    • Agent に複数ステップの作業を依頼し、その結果を安全に確認しながら取り込める
    • config.yaml を編集して、チームの開発スタイルに合わせた Continue の設定ができる
    • エアギャップ環境における外部通信の遮断設定と、その確認方法を説明できる

    いずれも、社内ネットワークの外へコードを送出することなく実現できます。

    初めて読む方へ

    はじめて Continue を使う方は、第 1 章から順番に読み進めることをお勧めします。特に第 4 章（外部通信の遮断確認）はセキュリティ上の重要手順です。スキップしないようにしてください。

    まとめ¶

    • Continue は VS Code 用の AI コーディングアシスタント拡張で、Chat・Autocomplete・Edit・Agent の 4 機能を提供する
    • 接続先 LLM を自由に選べるため、社内のローカル LLM に接続することでコードを社外に送出せずに AI 支援開発ができる
    • オンプレミス・エアギャップ環境での利用はセキュリティ・コンプライアンス上の要件を満たすうえで有効な選択肢となる
    • 本チュートリアルの対象は AI コーディング初心者の開発者で、VS Code・Continue・ローカル LLM の接続設定が完了していることを前提とする
    • AI の回答は常に自分で検証することが重要であり、ハルシネーション（誤情報の生成）には注意が必要である

    次の章へ¶

    次は 第 2 章 前提条件と環境確認 で、VS Code のバージョン確認やローカル LLM への疎通確認など、チュートリアルを進める前に確認すべき環境条件を整理します。
