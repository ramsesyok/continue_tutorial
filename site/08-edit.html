    第8章 Edit 機能でコードを書き換える

    この章で学ぶこと¶

    • Edit 機能が Chat・Autocomplete とどう違うのかを理解する
    • Ctrl+I（macOS: Cmd+I）でインライン編集を起動し、指示を入力する基本操作を身につける
    • 生成されたインライン差分（diff）を確認・承認・却下する方法を習得する
    • Ctrl+L（macOS: Cmd+L）による Chat への送信との使い分けを整理する
    • リファクタリングのハンズオンを通じて、実務での活用イメージをつかむ

    Edit 機能とは何か¶

    Edit（エディット）機能とは、エディタ上で選択したコードに対して自然言語で指示を与え、その場で書き換えを行う機能です。修正結果はサイドパネルではなくエディタの中にインライン差分として表示されるため、変更前後をひと目で比較し、承認か却下かを即座に判断できます。

    Autocomplete（第 7 章）との違いを整理しておきましょう。Autocomplete は「コードを書きながらリアルタイムで次の一手を提案する」機能です。一方 Edit は「すでに書いたコードを選んで明示的に書き換えを依頼する」機能です。書いた後で品質を上げる、いわば 後処理型の書き換えツール と理解してください。

    Chat（第 6 章）とはどう違うのでしょうか。Chat はサイドパネルで LLM と対話しながら情報を引き出したり、設計を相談したりするのに向いています。それに対して Edit は「このコードをこう直して」という 一発の書き換え指示 に特化しており、結果をそのままエディタに反映できる点が大きな違いです。

    観点 Edit Chat Autocomplete
    操作の起点 コードを選択して指示する テキストで質問・相談する コードを書いている最中
    結果の反映先 エディタ（インライン差分） サイドパネル（テキスト回答） エディタ（灰色テキスト候補）
    主な用途 リファクタリング・バグ修正・整形 設計相談・コード説明・調査 入力補完・スニペット生成
    確定操作 Accept / Reject 手動でコピー&ペースト Tab キー

    使い分けの指針

    「調べる・相談する」なら Chat、「今すぐ直す」なら Edit と覚えると迷いが少なくなります。Edit はエディタから手を離さずに修正を完結できるため、小回りの利くリファクタリングに特に向いています。

    操作の基本：Ctrl+I でインライン編集を起動する¶

    Edit の基本的な操作フローは次のとおりです。

    1. エディタ上で書き換えたいコードを選択する
    2. Ctrl+I（macOS: Cmd+I）を押す
    3. 画面上部に Edit 用の入力ボックスが表示される
    4. 指示を入力して Enter を押す
    5. 生成完了後、エディタ上に差分が表示される
    6. 差分を確認して Accept（承認）または Reject（却下）を選ぶ

    コードを選択せずに Ctrl+I を押すと、カーソルがある行の直下にコードを挿入するモードになります。既存コードの書き換えではなく新規コードを生成したいときに利用できます。

    キーバインドが変わっている場合

    Ctrl+I が他の操作に割り当て済みの場合は、VS Code のキーボードショートカット設定（Ctrl+K Ctrl+S）から「Continue: Edit」で検索し、任意のキーに変更できます。

    Edit ペインに指示を入力する¶

    入力ボックスには、変更したい内容を自然な日本語または英語で記述します。良い指示の書き方として次の点を意識してください。

    具体的に書く

    悪い例: 「改善して」
良い例: 「早期リターンを使ってネストを浅くしてください」

    一度に 1 つの変更を依頼する

    複数の変更をまとめて指示すると、モデルがどちらかを見落としたり、意図しない書き換えが起きたりすることがあります。「まず命名を直す」「次に型注釈を付ける」のように分けて実行するのが安全です。

    対象の範囲を選択範囲で明示する

    選択範囲が広いほどモデルが考慮すべきコードが増え、レスポンスが遅くなります。修正したい関数や条件分岐だけを選んで Edit を起動すると、速度と精度の両面で有利です。

    Edit に使用されるモデルは、config.yaml の models セクションで現在アクティブな Chat モデルです。補完専用モデル（tabAutocompleteModel）は Edit には使われません。Edit は比較的長いプロンプトを扱うため、Chat モデルと同じ高精度モデルが充てられる設計になっています。詳細は後述の「Edit とモデル設定：エアギャップ環境での確認」を参照してください。

    インライン差分（diff）を確認・操作する¶

    Edit の指示が完了すると、エディタ上に差分が表示されます。インライン差分（inline diff）とは、削除される行が赤のハイライト、追加される行が緑のハイライトで示される、変更前後の比較表示のことです。

    - def process(d):          ← 赤（削除される行）
-     for k in d:
-         if d[k] is None:
-             d[k] = ""
-     return d

+ def process(data: dict) -> dict:   ← 緑（追加される行）
+     for key, value in data.items():
+         if value is None:
+             data[key] = ""
+     return data

    差分はリアルタイムでストリームされてくるため、生成完了前でも内容を確認し始めることができます。

    変更を承認する（Accept）¶

    差分の内容が意図どおりであれば、次のいずれかで承認します。

    • 差分ブロックの上部に表示される 「Accept」 ボタンをクリックする
    • キーボードショートカット Ctrl+Shift+Enter（macOS: Cmd+Shift+Enter）を押す

    承認すると差分ハイライトが消え、書き換え後のコードがエディタに確定されます。

    変更を却下する（Reject）¶

    差分の内容が意図と違う、または品質が不十分な場合は却下します。

    • 差分ブロックの上部に表示される 「Reject」 ボタンをクリックする
    • キーボードショートカット Ctrl+Shift+Backspace（macOS: Cmd+Shift+Backspace）を押す

    却下すると書き換え前のコードに戻ります。指示を修正して再度 Ctrl+I を押し、別の指示でやり直してください。

    差分は確認してから承認する

    LLM が生成した差分は、意図どおりに見えても細部が変わっている場合があります。特に変数名の変更、条件式の微妙な変更、ロジックの省略などは目視でチェックしてから Accept することをおすすめします。

    Ctrl+L との違い：選択コードを Chat に送る場合¶

    コードを選択した状態で Ctrl+L（macOS: Cmd+L）を押すと、選択範囲が Chat のコンテキストとしてサイドパネルに送られます。書き換えはその場では行われず、Chat 上で「このコードについて〇〇を教えて」「このコードの問題点は何か」といった対話ができます。

    操作 ショートカット 用途
    Edit を起動する Ctrl+I（Cmd+I） 選択コードを今すぐ書き換える
    Chat に送る Ctrl+L（Cmd+L） 選択コードについて質問・相談する

    「まず Chat で問題を理解してから Edit で修正する」という 2 段階のフローが、慎重な修正が必要な場面では効果的です。

    Edit が活きるユースケース¶

    Edit は「既存コードを変換する」用途全般に活用できます。代表的なパターンを以下に示します。

    バグ修正・ロジック誤りの訂正

    怪しいと感じた関数を選択して「このループは正しく境界値を処理しているか確認し、問題があれば修正してください」と指示すると、LLM が問題を検出して書き換えます。ただし LLM の判断が常に正しいとは限らないため、差分の承認前に必ずロジックを確認してください。

    コードスタイルの統一

    「snake_case に統一してください」「PEP 8 に準拠した命名に変えてください」のように、コーディング規約への準拠を一括で依頼できます。

    docstring・コメントの自動生成

    関数を選択して「この関数に Google スタイルの docstring を追加してください」と指示すると、引数・戻り値・例外を含む説明文を自動生成します。

    型アノテーションの付与

    型注釈のない Python 関数を選択して「型ヒントを付けてください」と指示すると、引数と戻り値に適切な型が付与されます。

    処理の抽出・関数化

    長い関数の中から特定ブロックを選択して「この処理を独立した関数に抽出してください」と指示すると、新しい関数定義と呼び出し箇所への置き換えを自動で行います。

    Edit に向かない用途

    複数ファイルにわたる変更や、ファイルの新規作成・削除を伴うリファクタリングは Edit では扱えません。そのような大規模な変更には、第 9 章で扱う Agent 機能が適しています。

    ハンズオン：既存関数をリファクタリングする¶

    ここでは、読みにくいネストの深い Python 関数を Edit でリファクタリングします。config.yaml の設定が完了し、Chat モデルが社内サーバで動作していることを前提とします。

    題材のコード¶

    次のコードをファイル refactor_practice.py として VS Code で開いてください（または既存ファイルに貼り付けてください）。

    def calculate_discount(user, cart):
    if user is not None:
        if user.get("is_member"):
            if cart.get("total") is not None:
                if cart["total"] >= 10000:
                    discount = cart["total"] * 0.15
                else:
                    discount = cart["total"] * 0.05
            else:
                discount = 0
        else:
            if cart.get("total") is not None:
                discount = cart["total"] * 0.02
            else:
                discount = 0
    else:
        discount = 0
    return discount

    この関数は if のネストが 4 段になっており、ロジックの流れが追いにくい状態です。

    ステップ 1：コードを選択する¶

    関数全体（def calculate_discount から return discount まで）をマウスまたはキーボードで選択します。

    ステップ 2：Edit を起動する¶

    Ctrl+I（macOS: Cmd+I）を押します。エディタ上部に Edit の入力ボックスが表示されます。

    ステップ 3：指示を入力する¶

    次の指示を入力し、Enter を押します。

    早期リターン（ガード節）を使ってネストを浅くし、型ヒントも付けてください。

    ステップ 4：差分を確認する¶

    しばらくすると、エディタ上に差分が表示されます。以下のようなコードが生成されれば成功です。

    def calculate_discount(user: dict | None, cart: dict) -> float:
    if user is None:
        return 0

    total = cart.get("total")
    if total is None:
        return 0

    if user.get("is_member"):
        rate = 0.15 if total >= 10000 else 0.05
    else:
        rate = 0.02

    return total * rate

    元のコードと比べて、ネストが 1〜2 段に減り、ロジックが一直線に読めるようになりました。

    ステップ 5：差分を承認または却下する¶

    差分の内容が意図どおりであれば Accept（Ctrl+Shift+Enter）を押して確定します。もし期待と違った場合は Reject（Ctrl+Shift+Backspace）を押して戻し、指示を変えて再試行してください。

    例えば「ガード節の代わりに条件式（三項演算子）を使って 1 行にまとめてください」のように指示を変えると、異なるスタイルの差分が生成されます。何度でもやり直せるため、納得のいく形になるまで試してみましょう。

    ハンズオン後の確認

    リファクタリング後は、元の関数と同じ入出力になるかテストを実行して確認してください。LLM の書き換えは見た目上自然でも、まれに条件分岐の境界値が変わっていることがあります。

    Edit とモデル設定：エアギャップ環境での確認¶

    Edit が使うモデル¶

    Edit 機能は config.yaml の models セクションに設定された Chat モデルをそのまま利用します。補完専用モデル（tabAutocompleteModel）は Edit には使われません。

    現在アクティブなモデルは VS Code の右下ステータスバーで確認できます。複数モデルを設定している場合は、そこからモデルを切り替えることもできます。

    edit ロールを明示的に設定する（任意）¶

    config.yaml では models セクションの各モデルに roles を指定することで、どの機能にどのモデルを使うかを細かく制御できます。たとえば Chat には大型モデルを、Edit には処理速度の速い中型モデルを割り当てたい場合は次のように設定します。

    models:
  - title: Chat 用大型モデル
    provider: ollama
    model: <your-large-chat-model>
    apiBase: http://<your-internal-llm-host>:11434
    roles:
      - chat

  - title: Edit 用中型モデル
    provider: ollama
    model: <your-medium-model>
    apiBase: http://<your-internal-llm-host>:11434
    roles:
      - edit

    roles を省略した場合は、そのモデルが Chat・Edit の両方で使用されます。Edit のレスポンス速度が遅いと感じる場合は、roles でモデルを分離することを検討してください。

    補完専用モデルを Edit に使わない

    starcoder2 や deepseek-coder などの補完専用モデルは FIM（Fill-In-the-Middle）向けに最適化されており、長い自然言語の指示に応答する汎用 Chat 能力が低い場合があります。これらを roles: [edit] に指定すると、指示を正しく解釈した書き換えができないことがあるため注意してください。

    エアギャップ環境での動作確認¶

    Edit を初めて使う際は、次の手順でオフライン動作を確認しておきましょう。

    1. ネットワーク疎通確認: 社内 LLM サーバへの接続が正常か確認する（curl http://<your-internal-llm-host>:11434 等）
    2. 簡単な Edit を実行: 短いコードブロックを選択して「コメントを追加してください」と指示し、差分が生成されることを確認する
    3. 外部通信が発生していないことを確認: 第 4 章の手順に従い、プロキシログ等で外部アドレスへのリクエストが発生していないことをチェックする

    まとめ¶

    • Edit 機能は、選択したコードに対してその場で書き換えを行うインライン編集ツールです。Chat（相談・調査）や Autocomplete（入力補完）とは目的が異なります
    • Ctrl+I（macOS: Cmd+I）でインライン Edit を起動し、自然言語で指示を入力します。短く・具体的・1 変更 1 指示が良い指示の基本です
    • 生成されたインライン差分は赤（削除）・緑（追加）で表示されます。Ctrl+Shift+Enter で Accept、Ctrl+Shift+Backspace で Reject します
    • Ctrl+L（macOS: Cmd+L）はコードを Chat に送る操作です。「調べるなら Chat、直すなら Edit」と使い分けましょう
    • バグ修正・スタイル統一・docstring 生成・型アノテーション付与・関数抽出など、小〜中規模の書き換えで効果を発揮します
    • Edit が使うモデルは Chat モデルです。config.yaml の roles で Chat と Edit に別モデルを割り当てて速度を最適化することもできます

    次の章へ¶

    次は 第 9 章 Agent 機能を理解する で、より広い範囲を自律的に操作できる Agent 機能を扱います。Edit が「選択した箇所を直す」ツールであるのに対し、Agent は「目標を伝えると複数のファイルやツールを横断して実行する」機能です。エアギャップ環境で安全に使うための注意点とあわせて解説します。
