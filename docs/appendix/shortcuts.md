## この付録について

本付録は、Continue の各機能で使用するキーボードショートカットを一覧にまとめたリファレンスです。本文中に分散しているショートカット情報を 1 か所に集約しているため、操作に迷ったときや新しいメンバーへの参照資料として活用してください。

**OS 表記の規則**

本付録では、次の表記で OS を区別します。

| 表記 | 対象 OS |
| --- | --- |
| `Cmd` | macOS |
| `Ctrl` | Windows / Linux |

macOS と Windows / Linux でキーが異なる場合は、それぞれ別の列に記載します。`Tab` や `Esc` など OS によらず共通のキーは 1 列で示します。

## Chat のショートカット

Chat パネルを使った基本的な対話操作のショートカットです。詳細な機能説明は [第 6 章](../06-chat-basics.md) を参照してください。

### Chat パネルの開閉・切替

| 操作 | Windows / Linux | macOS |
| --- | --- | --- |
| Chat パネルを開く / 閉じる | `Ctrl+L` | `Cmd+L` |
| 新しい会話を開始する | `Ctrl+Shift+L` | `Cmd+Shift+L` |

`Ctrl+L`（macOS: `Cmd+L`）は Chat パネルのトグルとして機能します。エディタで作業中にパネルを素早く表示・非表示にしたいときに使います。

新しい会話（`Ctrl+Shift+L` / `Cmd+Shift+L`）は、別のトピックに移るときや、現在の会話の文脈をリセットしたいときに使います。過去の会話履歴は Chat パネル上部の履歴ボタンから確認できるため、必要な情報が失われることはありません。

!!! tip "コードを選択してから開く"
    エディタでコードを選択した状態で `Ctrl+L`（macOS: `Cmd+L`）を押すと、選択範囲が Chat のコンテキストとして自動的に添付されます。「このコードについて質問する」という操作が 1 アクションで完了します。

### コンテキスト追加操作

Chat の入力欄でコンテキストを指定する操作は、キーボード入力で完結します。専用のショートカットキーではなく、入力欄に `@` を打ち込む操作が起点です。

| 操作 | 入力 |
| --- | --- |
| コンテキストプロバイダのメニューを開く | 入力欄に `@` を入力する |
| ファイルを指定する | `@file` と入力してファイルを選択する |
| リポジトリ全体を対象にする | `@codebase` と入力する |
| ターミナル出力を参照する | `@terminal` と入力する |
| 「問題」パネルの内容を参照する | `@problems` と入力する |
| 開いているすべてのタブを対象にする | `@open` と入力する |

メニューが表示された後は、矢印キーで候補を選び `Enter` で確定します。コンテキストプロバイダの詳しい使い方は [第 6 章](../06-chat-basics.md) を参照してください。

### メッセージ送信・履歴操作

| 操作 | キー |
| --- | --- |
| メッセージを送信する | `Enter` |
| 入力欄で改行する（送信しない） | `Shift+Enter` |

`Enter` で送信、`Shift+Enter` で改行という 2 つのキーを使い分けることで、複数行にわたる質問も入力欄内で整理してから送れます。

## Autocomplete（タブ補完）のショートカット

コードを入力しながら使うタブ補完の操作ショートカットです。機能の仕組みや設定については [第 7 章](../07-autocomplete.md) を参照してください。

### 補完の承認・却下

| 操作 | キー |
| --- | --- |
| 補完候補を確定する（全体） | `Tab` |
| 補完候補を却下する | `Esc` |

補完候補が薄いグレーで表示されたら、`Tab` キーで全体を一括確定します。候補が不要な場合は `Esc` を押すと非表示になり、入力を続けられます。

### 補完の強制起動

補完候補が自動で表示されない場合や、表示が遅いと感じる場面で、手動でリクエストを送るショートカットです。

| 操作 | Windows / Linux | macOS |
| --- | --- | --- |
| 補完を手動でトリガーする | `Alt+\` | `Option+\` |

!!! note "補完が表示されないときの確認手順"
    手動トリガーを押しても候補が出てこない場合は、VS Code 右下のステータスバーにある Continue のアイコンを確認してください。エラー表示が出ているときはモデルへの接続に問題がある可能性があります。[第 14 章](../14-troubleshooting.md) のトラブルシューティングを参照してください。

## Edit のショートカット

選択したコードをインラインで書き換える Edit 機能のショートカットです。機能の概要とハンズオンは [第 8 章](../08-edit.md) を参照してください。

### Edit モードの起動

| 操作 | Windows / Linux | macOS |
| --- | --- | --- |
| Edit を起動する（選択範囲を書き換え） | `Ctrl+I` | `Cmd+I` |
| 選択コードを Chat に送る | `Ctrl+L` | `Cmd+L` |

`Ctrl+I`（macOS: `Cmd+I`）は、エディタ上でコードを選択した状態で押すと Edit の入力ボックスが表示されます。コードを選択せずに押した場合は、カーソル行の直下にコードを**挿入**するモードになります。

`Ctrl+L`（macOS: `Cmd+L`）は、コードを選択した状態で押すと選択範囲を Chat のコンテキストとして送信します。「直す前にまず Chat で相談する」場合に使います。

!!! tip "Edit と Chat の使い分け"
    「今すぐコードを書き換えたい」ときは `Ctrl+I`（`Cmd+I`）で Edit を起動します。「書き換える前に内容を確認したい・相談したい」ときは `Ctrl+L`（`Cmd+L`）で Chat に送ります。2 つのキーの使い分けがそのまま Edit と Chat の使い分けに対応しています。

### 変更の承認・却下

Edit の実行後に表示されるインライン差分（diff）を確定または取り消す操作です。

| 操作 | Windows / Linux | macOS |
| --- | --- | --- |
| 変更を承認する（Accept） | `Ctrl+Shift+Enter` | `Cmd+Shift+Enter` |
| 変更を却下する（Reject） | `Ctrl+Shift+Backspace` | `Cmd+Shift+Backspace` |

差分が表示されたら、内容を目視で確認してから承認または却下を選びます。却下すると書き換え前のコードに戻るため、指示を変えて `Ctrl+I`（`Cmd+I`）でやり直せます。

!!! warning "承認前に差分を確認する"
    LLM の書き換えは意図どおりに見えても、変数名の微妙な変更やロジックの省略が含まれることがあります。Accept する前に差分の削除行（赤）と追加行（緑）を必ず確認してください。

## Agent のショートカット

Agent 機能は、Chat パネルの入力欄近くにあるモード切替セレクタから `Agent` を選択することで起動します。Chat との切替はこのセレクタを使った操作が基本で、デフォルトの Continue 設定では Agent 起動専用のキーボードショートカットは割り当てられていません。

### Agent の実行制御

Agent の実行中に役立つ操作を示します。いずれもサイドパネル上の UI 操作です。

| 操作 | 方法 |
| --- | --- |
| ツール呼び出しを承認する（Accept） | 承認ダイアログの「Accept」ボタンをクリック |
| ツール呼び出しを却下する（Reject） | 承認ダイアログの「Reject」ボタンをクリック |
| Agent の実行を中断する | サイドパネルの停止ボタン（■）をクリック |

承認ダイアログは、Agent がファイルの編集・コマンドの実行などを行おうとするたびに表示されます。ダイアログが表示されたら、操作の対象・内容・副作用を確認してから判断してください。詳細な確認手順は [第 9 章](../09-agent.md) を参照してください。

!!! tip "Agent モードに専用ショートカットを割り当てたい場合"
    VS Code のキーボードショートカット設定（`Ctrl+K Ctrl+S` / `Cmd+K Cmd+S`）を開き、`continue` で検索すると Continue のコマンド一覧が表示されます。Agent モード切替のコマンドが表示された場合は、任意のキーを割り当てられます。カスタマイズの具体的な手順は「[ショートカットのカスタマイズ方法](#ショートカットのカスタマイズ方法)」を参照してください。

本付録に登場したショートカットをすべて 1 か所に集約した早見表です。チームへの配布や、手元に置いて参照するための資料としてご利用ください。

| **Chat** | 新しい会話を開始する | `Ctrl+Shift+L` | `Cmd+Shift+L` |
| **Chat** | メッセージを送信する | `Enter` | `Enter` |
| **Chat** | 入力欄で改行する | `Shift+Enter` | `Shift+Enter` |
| **Autocomplete** | 補完候補を確定する | `Tab` | `Tab` |
| **Autocomplete** | 補完候補を却下する | `Esc` | `Esc` |
| **Autocomplete** | 補完を手動でトリガーする | `Alt+\` | `Option+\` |
| **Edit** | Edit を起動する | `Ctrl+I` | `Cmd+I` |
| **Edit** | 選択コードを Chat に送る | `Ctrl+L` | `Cmd+L` |

    Continue のバージョンアップに伴い、デフォルトのキーバインドが変更されることがあります。上表のショートカットが動作しない場合は、VS Code のキーボードショートカット設定（`Ctrl+K Ctrl+S` / `Cmd+K Cmd+S`）を開き、`continue` で検索して現在の割り当てを確認してください。

## ショートカットのカスタマイズ方法

Continue のすべてのコマンドは、VS Code の標準的なキーバインド変更機能でカスタマイズできます。デフォルトの割り当てが他の拡張機能と競合している場合や、より押しやすいキーに変更したい場合に活用してください。

### キーボードショートカット設定を開く

1. `Ctrl+K Ctrl+S`（macOS: `Cmd+K Cmd+S`）を押す（または [ファイル] → [ユーザー設定] → [キーボードショートカット]）
2. 検索欄に `continue` と入力すると、Continue に関連するコマンドが絞り込まれて表示される
3. 変更したいコマンドをダブルクリックし、割り当てたいキーを押す

### `keybindings.json` で直接編集する

GUI での変更は、裏側で `keybindings.json`（ユーザー設定ファイル）に書き込まれます。ファイルを直接編集する場合は、キーボードショートカット画面の右上にある「`{}`（JSON を開く）」ボタンをクリックして開きます。

以下は Edit（`Ctrl+I`）を別のキーに再割り当てする場合の記述例です。

```json
[
  {
    "key": "ctrl+shift+e",
    "command": "continue.acceptDiff",
    "when": "editorTextFocus"
  }
]
```

- `key`: 割り当てたいキーの組み合わせ（Windows / Linux 形式で記述する。macOS では自動的に読み替えられる）
- `command`: Continue のコマンド ID（キーボードショートカット設定画面で `continue` 検索して確認する）
- `when`: キーが有効になる条件（`editorTextFocus` はエディタにフォーカスがある状態）

!!! tip "競合を確認する"
    新しいキーを割り当てる前に、同じキーが他のコマンドに割り当てられていないか確認してください。キーボードショートカット設定画面では、割り当て済みのコマンドが一覧に表示されます。競合がある場合は、既存の割り当てを削除するか、別のキーを選んでください。

!!! note "エアギャップ環境での設定の共有"
    チームで設定を共有する場合は、`keybindings.json` の内容を社内 Git リポジトリや共有ドライブに保管し、各メンバーの VS Code プロファイルに配布する運用が便利です。詳細な VS Code 設定の配布方法は [第 13 章](../13-security-operations.md) を参照してください。

## まとめ

- Chat は `Ctrl+L`（macOS: `Cmd+L`）でパネルを開き、`Ctrl+Shift+L`（macOS: `Cmd+Shift+L`）で新しい会話を始める
- Autocomplete は `Tab` で確定、`Esc` で却下、`Alt+\`（macOS: `Option+\`）で手動トリガーという 3 つのキーが基本操作
- Edit は `Ctrl+I`（macOS: `Cmd+I`）で起動し、差分を確認後に `Ctrl+Shift+Enter` で Accept、`Ctrl+Shift+Backspace` で Reject する
- Agent のモード切替は UI セレクタで行い、デフォルトでは専用のキーボードショートカットは割り当てられていない
- `Ctrl+K Ctrl+S`（macOS: `Cmd+K Cmd+S`）で VS Code のショートカット設定を開き、Continue のコマンドを自由に再割り当てできる
