## ようこそ

本ドキュメントは、[Continue.dev](https://www.continue.dev/) の VS Code 拡張を、完全なオンプレミス・エアギャップ環境のローカル LLM と組み合わせて利用するための、ソフトウェア開発者向けチュートリアルです。

GitHub Copilot などの AI コーディング支援ツールを初めて利用する開発者でも、Continue を業務開発に組み込めるようになることを目指します。

## 対象読者

- ソフトウェア開発者（業務でコードを書く方）
- AI コーディング支援ツール（Copilot 等）の利用経験は問いません
- VS Code を日常的に利用している、または利用予定の方

## 前提環境

- **IDE**: VS Code（本書では JetBrains 系 IDE は対象外）
- **ネットワーク**: 完全オンプレミス・エアギャップ環境（外部インターネットへの到達不可）
- **LLM**: 社内に配置されたローカル LLM（Ollama / vLLM / LM Studio など）
- **Continue とローカル LLM の接続設定は完了している**ことを前提とします

!!! note "接続設定が未完了の場合"
    Continue からローカル LLM への接続設定がまだの場合は、本書を読み始める前に社内の LLM 基盤チーム提供の接続手順を参照し、最低限 Chat 経由でモデル応答が得られる状態にしてください。

## 本書の使い方

- **初めて Continue に触れる方**: 第 1 章から順に読み進めてください。とくに第 3 章までで導入を完了させ、第 4 章で外部通信が発生しないことを確認してから本格的に利用を開始することを推奨します。
- **既に Continue を試用したことがある方**: 重点章（第 4・7・8・9・10・11 章）から拾い読みしても理解できる構成にしています。
- **運用・監査担当の方**: 第 4 章（テレメトリ遮断確認）と第 13 章（セキュリティと運用）を中心に参照してください。

## エアギャップ前提について

本書のすべての手順は、**外部インターネットへの接続が一切できない環境**で完結することを前提に書かれています。

- 外部 URL への接続を必要とする手順は記載しません
- 外部からのパッケージ取得が必要な場合は、社内ミラー等で**事前にオフライン配布されている前提**で説明します
- クラウド側のサービス（リモート埋め込み API 等）に依存する Continue の機能には、必要に応じて言及しますが**本書の構成では使用しません**
