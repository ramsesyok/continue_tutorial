# 執筆進捗管理

本ファイルは、各章の執筆状況を一元管理するための台帳です。新しいセッションを開始するエージェントは、まず本ファイルを参照し、次に着手すべき章と現在の状態を確認してください。

!!! note "編集時の注意"
    本ファイルは複数セッションから書き換えられる可能性があります。編集する前に必ず Read で最新内容を取得し、自分のセッションで触る行のみを更新してください。

---

## 1. ステータス定義

| ステータス | 意味 |
| --- | --- |
| 未着手 | まだ作業を開始していない |
| 節構成案作成中 | エージェントが節構成案を作成中 |
| 節構成承認待ち | 節構成案をユーザーが確認中 |
| 節構成承認済 | 節構成が確定し、本文執筆に進める |

| セルフチェック中 | 執筆後のセルフチェックを実施中 |
| レビュー待ち | ユーザーレビュー中 |
| 修正中 | レビュー指摘事項の反映中 |
| 完了 | レビューが完了し本書に組み込み済 |

## 3. 更新ルール

- ステータスを変更したエージェントは、必ず「最終更新」列に日付（`YYYY-MM-DD` 形式）を記入する
- 「メモ」列には特記事項（詰まっている点、レビュー指摘の要点、依存している他章など）を簡潔に書く
- 編集の前に必ず Read で最新状態を取得する（他セッションが先に更新している可能性があるため）
- ステータスが「完了」になった章は、以降のセッションで内容を書き換えない（必要な修正は新しい指摘として `5. 指摘事項管理` に記録する）
- 章別ガイドライン上の特記事項（重点章のハンズオン等）が満たされていない場合は、ステータスを「完了」にせず「修正中」に戻す

## 6. セッション開始時の確認手順

新しいセッションを開始するエージェントは、次の順序で確認してください。

1. 本ファイル（`PROGRESS.md`）を読み、次に着手すべき章と現在のステータスを確認する
2. `AGENT_INSTRUCTIONS.md` を読み、執筆ルールを確認する
3. 自分が担当する章のステータスを適切に更新する（例: `未着手` → `節構成案作成中`）
4. 必要に応じて前後章のファイルを Read し、整合を確認する
5. 執筆 / レビューに着手する
